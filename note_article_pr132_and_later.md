# ゆかりリクエストWebツール（KaraokeRequestorWeb）最新アップデートまとめ（#132以降）

## はじめに

持ち込みカラオケ用Webリクエストツール「ゆかりリクエストWebツール」に、2026年4月〜5月にかけて大きなアップデートが追加されました。この記事では **PR #132〜#136** の変更内容と設定方法をまとめます。

---

## PR #132：マイページ機能の追加・Google Drive同期

### 概要

マイページが大幅に強化されました。ユーザーの選曲履歴や後で歌うリスト、お気に入り曲などをブラウザをまたいで引き継げるようになりました。さらに**Google Driveへのデータ同期**機能が追加され、複数サーバー間でデータを共有できるようになっています。

### 新機能一覧

#### マイページ基本機能
- Cookieによるユーザー識別
- 表示名の設定・変更
- プロフィールアイコンのアップロード（JPEG/PNG/GIF/SVG/WebP）

#### 選曲履歴
- リクエスト時に自動記録
- 日付順・回数順での並び替え・削除

#### 後で歌うリスト / お気に入り曲
- 曲の追加・削除・リクエストへのショートカット
- ListerDB連携でファイルが別フォルダに移動しても対応

#### お気に入り検索ワード
- 検索結果画面から「検索ワードを保存」で登録
- 検索種別・部分一致設定も保存

#### インポート / エクスポート
- 全マイページデータをJSONファイルとしてエクスポート
- 追加（マージ）／上書きの2モードでインポート

#### デバイスリンク
- ペアリングコードで別端末からの引き継ぎに対応

#### Google Drive同期（新機能）
- Googleアカウントと連携してデータをDrive appdataに保存
- 複数サーバー間でデータを共有可能
- 「DriveのデータをMerge」するだけで別サーバーへの引き継ぎが完了

### Google Drive同期の設定方法

Google Drive同期を使うには、**中継サーバー（ykr.moe）を利用する方法**と**自前サーバーを立てる方法**があります。

#### 通常の利用（ykr.moe中継サーバーを使う場合）

Google Cloud Consoleの設定はすでに済んでいます。設定画面（`init.php`）の **「Google同期設定」** セクションに以下を入力して保存するだけで利用できます：

- **Google Client ID**：サポートDiscordで入手したクライアントID
- **Google Client Secret**：空欄のままでOK
- **中継サーバーURL**：デフォルト `https://ykr.moe/mypage_google_callback.php` のまま
- **中継シークレット**：サポートDiscordで入手したシークレット

Client IDや中継シークレットの入手方法など、設定に関する質問はサポートDiscordでお気軽にどうぞ：
**https://discord.gg/9wtZApZU5P**

#### 自前中継サーバーを立てる場合

自分でOAuth中継サーバーを立てる場合は、まずGoogle Cloud Consoleで設定が必要です。

**Google Cloud Consoleの設定手順：**

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「OAuth 2.0 クライアント ID」を作成（アプリケーション種類：Webアプリケーション）
3. リダイレクトURIに自前サーバーのコールバックURL（例：`https://yourserver.example.com/mypage_google_callback.php`）を登録
4. 取得したクライアントIDとシークレットを控えておく

**中継サーバーの設置手順：**

1. `mypage_google_relay_server.php` を自分のサーバーに `mypage_google_callback.php` としてリネームして配置
2. 同じ場所に `mypage_google_relay_config.php` を設置し、クライアントIDとシークレットを記載

```php
$RELAY_CLIENT_ID     = 'YOUR_CLIENT_ID.apps.googleusercontent.com';
$RELAY_CLIENT_SECRET = 'YOUR_CLIENT_SECRET';
$RELAY_SECRET        = '32文字以上のランダム文字列'; // openssl rand -hex 32 で生成
```

3. 設定画面の「Google同期設定」に以下を入力して保存：
   - **Google Client ID**：取得したクライアントID
   - **Google Client Secret**：取得したクライアントシークレット
   - **中継サーバーURL**：自前サーバーのコールバックURL
   - **中継シークレット**：`mypage_google_relay_config.php`の`$RELAY_SECRET`と同じ値

## PR #133：シークレット予約の表示バグ修正

### 変更内容

予約一覧のスワイプ画面（`requestlist_swipe.php`）移行前は正常に動作していた、シークレット曲の表示ロジックのバグを修正しました。

**修正内容：** シークレット（秘密）設定の曲について、再生済みの場合はタイトルを表示するよう修正しました。

以前は`secret`フラグが立っている曲が常にタイトルをマスク表示（先頭1文字＋アスタリスク）していましたが、正しいロジックに戻しました：

- **未再生 or 初期状態（`nowplaying` が「未再生」または `1`）** → マスク表示
- **再生済みの場合** → タイトルを表示

これにより、歌い終わった後にその曲が何だったかを予約一覧で確認できるようになります。

## PR #134：検索画面UI/UX刷新・ナビバー統一・各ページBootstrap 5化

### 概要

検索画面が**Bootstrap 5ベース**で全面リニューアルされました。また、BS3のページとBS5のページでナビバーの見た目が統一されています。

### 検索UIの主な改善点

| 改善項目 | 内容 |
|---|---|
| スティッキーサブメニューバー | スクロールしても検索タブが画面上部に追従 |
| パンくずリスト | 現在位置が分かりやすく表示される |
| ページネーション | 前後ボタンに加えてページ番号ボタンを追加（スライドウィンドウ表示） |
| ファイル一覧のカード表示 | ファイルがカードスタイルで見やすく表示 |
| テーマCSS変数 | デザインのカスタマイズが容易に |

### 新検索UIの有効化方法

設定画面の **「新デザイン検索画面UI（Bootstrap 5）」** から「使用する」に変更して保存すると有効になります。

有効にすると、既存の検索ページへのアクセスは自動的にBS5版にリダイレクトされます。直接 `search_bs5.php` にアクセスすることも可能です。

### ナビバーの高さ統一

BS3ページのナビバー高さを、新しく実装したBS5ページと統一して **56px** に調整しました。

- スマートフォンでのバーガーメニュー展開時も正しく表示
- 「管理者ログイン中」ラベルの余白・フォントサイズも調整
- ナビバーブレークポイントを992px→768px（`navbar-expand-md`）に変更

### BS5化されたページ

- `requestlist_swipe.php`（予約一覧スワイプ画面）
- `mypage.php`（マイページ）

## PR #135：制作者別音量増減機能の追加

### 概要

既存の「制作者別音ズレ設定」に加えて、**制作者ごとの音量増減設定**ができるようになりました。動画によって音量が大きすぎたり小さすぎたりする場合に、自動で補正されます。

### 仕様

- 設定値は **−100〜+100（整数）** で、相対的な増減を表します
- `0`は「変更なし」（全体設定の音量動作をそのまま使用）
- 適用音量 = `全体の基準音量 + オフセット値` を 0〜100 にクランプして適用
- サーバーによって基準音量が異なっても、相対値なので一貫して機能します

### 設定方法

1. 管理画面の **「制作者別音ズレ・音量初期値設定」** を開く
2. 制作者ルールの追加フォームに「音量増減」欄が追加されているので、増減したい値を入力
3. 予約一覧カードに「音量：+10%」のように表示されます

また、予約確認画面でも個別に音量を調整できます：
- **「−5%」「+5%」「±0に戻す」** ボタンで調整可能
- 差し替え予約の場合は元の予約の音量設定が引き継がれます

## PR #136：リクエスト確認画面で歌う人が誤選択されるバグ修正

### バグの概要

同じWiFi（NAT環境）で同じブラウザを使う複数人が、お互いの名前が選ばれてしまうバグが修正されました。

### バグの原因

`YkariUsername`クッキーが未設定のユーザーを「IPアドレス＋User-Agent」で識別していたため、以下の条件が重なると誤動作していました：

- 同じWiFiを使う複数人 → 公開IPアドレスが一致
- 同じブラウザを使う → User-Agentが一致

その結果、別の人の名前が自動選択された状態になってしまっていました。

### 修正内容

リクエスト送信（`exec.php`）成功後に `YkariUsername` クッキーを自動設定するように変更。次回以降のリクエスト確認画面では、IP+UAマッチングより優先してクッキーの名前が選ばれます。

**注意事項：**
- 初回リクエスト前（クッキー未設定）は従来通りIP+UAマッチングが使われます
- 共有端末モード（`blank_username = true`）の場合は影響なし
- 「小休止」リクエスト時はクッキーを更新しません

## アップデート手順

設定画面にリンクがある **「オンラインアップデート」** 画面から実施できます。

- データベースのマイグレーションは自動的に行われます（`requesttable`への`volume`列追加など）
- Google Drive同期を使う場合は別途、設定画面の「Google同期設定」で設定が必要（前述の設定方法を参照）
- 新検索UIを使う場合は設定画面の **「新デザイン検索画面UI（Bootstrap 5）」** から「使用する」に変更
